# Resource Inclusion Criteria

## Include A Resource If

- it teaches a reusable workflow
- it is useful for economics or finance research
- it is official documentation for an important tool
- it gives working code or a reproducible example
- it explains risks, limits, or failures
- it helps scholars build skills rather than chase tools

## Exclude Or Mark Cautiously If

- it is generic AI hype
- it gives unsupported tool rankings
- it claims to write a paper in one hour
- it shares unverified prompts
- it makes uncited claims about tool performance
- it ignores data privacy or Git
- it discourages verification
