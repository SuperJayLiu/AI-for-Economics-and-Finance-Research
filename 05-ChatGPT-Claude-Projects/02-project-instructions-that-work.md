# Project Instructions That Work

Good project instructions are specific enough to shape behavior and short enough to maintain.

## Recommended Instruction Blocks

Purpose:

- what the project is for
- what it is not for

Evidence rules:

- never invent citations
- cite only uploaded or verified sources
- flag unsupported claims

Research standards:

- separate model, data, identification, results, limitations, and contribution
- avoid overstating causal interpretation
- preserve equations, notation, and empirical terms

Verification:

- list claims requiring manual checking
- identify source locations when possible
- state uncertainty clearly

Output:

- use structured headings
- include checklists
- keep drafts concise unless asked otherwise
