# Empirical Paper Project

## Purpose

Support an applied economics or finance project from research design through coding, interpretation, and writing.

## Best For

- explaining empirical strategy
- reviewing code
- planning variable construction
- checking table logic
- drafting results interpretation
- identifying robustness questions

## Not Suitable For

- choosing identification strategy without expert judgment
- fabricating results
- interpreting results without seeing actual outputs
- uploading restricted or proprietary data to public tools

## What To Upload

- research question
- nonconfidential data dictionary
- code snippets or scripts you are allowed to share
- table shells or regression output
- notes on sample construction
- institutional background that is safe to share

## What Not To Upload

- restricted administrative data
- licensed commercial data if upload is prohibited
- confidential firm or transaction data
- proprietary code without permission
- raw data containing personal or sensitive information

## Project Instructions

- Treat empirical design as a scholar-controlled decision.
- Separate code explanation from causal interpretation.
- Flag possible threats to identification.
- Never invent coefficients, tables, robustness checks, or sample sizes.
- When reviewing code, identify assumptions and possible failure points.
- Ask for actual outputs before interpreting results.

## Recommended Workflow

1. Describe the research question and unit of observation.
2. Map variables, sample restrictions, and data sources.
3. Ask AI to explain or review code in small pieces.
4. Run code yourself.
5. Compare outputs to expected table shells.
6. Ask for interpretation caveats.
7. Verify economic magnitude, not only statistical significance.
8. Record all AI-assisted code or writing changes.

## What The AI Should Produce

- variable construction checklists
- code review notes
- table interpretation drafts
- robustness planning questions
- lists of assumptions and possible threats

## What The Scholar Must Verify

- code execution
- data transformations
- sample restrictions
- regression specification
- table construction
- causal interpretation
- license and privacy constraints

## Common Mistakes

- uploading data before checking permissions
- trusting generated code because it looks correct
- asking AI to "find identification"
- writing causal language stronger than the design permits

## Disclosure Notes

Track whether AI assisted with code, debugging, writing, or interpretation. Keep enough detail to explain what the human researcher verified.
