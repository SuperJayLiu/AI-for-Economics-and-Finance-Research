# Tool Claims Must Be Dated

> [!WARNING]
> Any sentence like "Claude is best," "Codex is best," or "Cursor is best" starts expiring the moment it is written.

## What Problem This Solves

Claims about ChatGPT, Claude, Codex, Cursor, Copilot, MCPs, and agent tools become outdated quickly.

For scholars, outdated tool advice can waste time or create risk. A tool that is good for coding today may be weak for literature review, expensive for long projects, unsafe for restricted data, or replaced by a better workflow next month.

## Required Format For Tool Claims

Use this block on tool-comparison pages:

```text
Last checked:
Tool versions/plans checked:
Author's judgment:
Known uncertainty:
```

## Better Tool Claim Template

```text
For this task:
Tool tested:
Date checked:
Plan/account type:
Input tested:
Output quality:
Verification burden:
Privacy/data concern:
Cost or usage limit:
Would use again for:
Would not use for:
```

## Rule

Do not write timeless claims such as "Tool X is best." Write dated claims tied to specific tasks.

Better:

```text
As of 2026-05-24, this tool was useful for this task under these assumptions.
```

## Examples

| Weak claim | Better claim |
| --- | --- |
| Codex is best for research | As of 2026-05-24, Codex worked well for reviewing this Python replication script after the repo had an `AGENTS.md` file and tests. |
| Claude is best for writing | As of 2026-05-24, Claude was useful for restructuring this introduction, but all claims and citations were checked manually. |
| Use MCP for everything | MCP was useful for connecting to this source library, but it increased permission and data-exposure risk. |

> [!TIP]
> Compare tools by research task, not by brand. The useful question is: "Which tool improves this workflow while keeping verification manageable?"

## Sources and Workflow Influences

Draws on public tool-comparison practice, official tool documentation, and the rapid change rate in AI research tools.
