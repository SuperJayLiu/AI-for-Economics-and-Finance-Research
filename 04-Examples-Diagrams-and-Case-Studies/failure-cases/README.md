# Failure Cases

AI failures often look plausible. Use these examples to learn how to detect and prevent mistakes.

## Case Format

Each case should include:

- what happened
- why it looked plausible
- what went wrong
- how to detect it
- how to prevent it

## Planned Cases

- fake citation generated by AI
- wrong Stata code that runs
- wrong interpretation of regression coefficient
- AI overwrote raw data
- AI misunderstood event-study timing
- AI produced beautiful but misleading figure
- AI overclaimed causal evidence
- AI invented contribution relative to literature
- AI created a factor-mining story

## First Rule

The more polished the AI output looks, the more disciplined the verification should be.
