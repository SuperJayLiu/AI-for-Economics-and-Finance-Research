# AI Research Update Digest Workflow

Use this to stay updated without chasing tool hype.

> [!NOTE]
> Default add-on for this workflow: `If any required input, source type, date, tool term, permission boundary, or output format is unclear, ask me up to five clarifying questions before acting. Define unfamiliar technical terms in plain language and end with "Questions for you" if anything remains uncertain.`

## Principle

Follow builders, official docs, and field users. Extract workflows, not opinions.

## Copy-Paste Workflow

```text
Build a weekly AI-for-econ-finance research update digest for me.

Audience:
Economics and finance researchers who care about research workflows, coding, writing, data safety, GitHub, skills, agents, and reproducibility.

Sources to prioritize:
1. Official docs for tools I use: [OpenAI/Codex, Anthropic/Claude, GitHub, MCP, etc.]
2. Economists and finance researchers showing real workflows.
3. Builders who publish working skills, code, or agent setups.
4. GitHub repos with reproducible examples.
5. Academic papers on AI and research integrity.

Do not prioritize:
- viral tool lists
- unsupported claims that one model is always best
- "write a paper in one hour" claims
- advice that ignores data privacy or Git

For each item, return:
- title
- source and link
- why it matters for economics/finance research
- what workflow idea can be extracted
- whether it is beginner/intermediate/advanced
- risks or caveats
- last checked date

End with:
1. Three things worth trying this week.
2. Three things to ignore for now.
3. One skill/template that should be added or updated.
```
