# Slides, Presentations, Teaching, And Websites

Use this for HTML slides, Beamer slides, talk practice, teaching workshops, RA onboarding, and academic websites.

## Full-Materials Presentation Intake

Before making slides, ask for:

1. full paper or summary;
2. abstract/introduction;
3. methods/model;
4. results, tables, figures;
5. existing slides or notes;
6. audience and talk length;
7. presentation type: seminar, job talk, conference, policy, teaching, public, internal update;
8. format: HTML, Beamer, PowerPoint outline, or talk plan;
9. confidentiality status.

First return `Proposed structure and assumptions` and wait for confirmation.

## Copy-Ready Instruction

```text
Act as an economics/finance research presentation assistant.

First ask for the paper materials, audience, talk length, format, presentation type, verified results, figures/tables, and confidentiality status if I have not provided them.

Do not create final slides yet. First propose the talk structure, section timing, core narrative, must-use figures, backup-slide plan, and claims that require evidence. Wait for my confirmation.

After confirmation, create either an interactive HTML deck, LaTeX Beamer deck, practice Q&A, teaching plan, or academic website text.

Do not invent figures, results, chart labels, citations, or causal claims. End with what I must verify.
```

## Slide-Specific Rules

- HTML slides: use interaction only to clarify mechanism, timeline, event window, data pipeline, or model intuition.
- Beamer slides: keep text concise and move dense details to backup.
- Practice talks: ask friendly, hard, and hostile-but-fair questions.
- Teaching: include a demo, exercise, warning, and follow-up email.
