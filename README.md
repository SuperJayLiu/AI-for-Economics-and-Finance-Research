## English

This is a practical GitHub-native handbook and resource library for using AI responsibly in economics and finance research.

This handbook is not a prompt collection and not a list of AI tools.

It teaches economics and finance scholars how to build reliable AI-assisted research workflows: how to use AI, when not to use it, how to verify outputs, how to protect data, how to organize projects, and how to keep up with fast-changing tools.

### Who This Is For

- PhD students and research assistants
- Junior faculty and applied economists
- Finance researchers and policy researchers
- Instructors building AI research workshops
- Research teams that need shared rules for AI-assisted work

### Core Rule

AI can automate labor, but not scholarly responsibility.

For serious work, use one GitHub repository for the research project, one ChatGPT or Claude Project for the AI workspace, one AI-use log for traceability, and one clear folder structure for code, data, paper, and results.

### Click, Copy, And Use

The repo is designed so readers can click into a topic, copy a reusable instruction, paste it into ChatGPT/Claude/Codex/Cursor, and adapt it to their own research project. Pages should be readable by nontechnical scholars and should make inputs, outputs, risks, and verification steps explicit.

### How This Guide Uses External Resources

This guide draws on public work by economists, finance scholars, AI workflow builders, and official tool documentation. It does not simply collect links. It extracts reusable workflow patterns, cites sources briefly where relevant, and adapts the ideas into economics and finance research instructions.

Tool behavior changes quickly, so tool-specific notes should be dated and periodically reviewed.

### What AI Can Help With

- organizing literature notes
- clarifying research questions and mechanisms
- explaining and reviewing Stata, R, Python, MATLAB, and LaTeX
- improving paper structure and language
- preparing seminar talks and referee responses
- building repeatable project workflows

### What AI Should Not Be Trusted With

- citations without verification
- causal interpretation without expert judgment
- confidential data or referee material without policy approval
- final literature claims
- theoretical derivations without manual checking
- final submission decisions

## 中文

这是一个 GitHub 原生的手册和资源库，帮助经济学和金融学研究者负责任地使用 AI 和大语言模型。

本仓库不是“提示词合集”，也不是“AI 工具清单”。

它关注的是如何建立可靠的 AI 辅助研究工作流：什么时候可以用 AI，什么时候不应该用，如何核查输出，如何保护数据，如何组织研究项目，以及如何跟上快速变化的 AI 工具。

### 适合谁阅读

- 博士生和研究助理
- 青年教师和应用经济学研究者
- 金融学研究者和政策研究者
- 希望设计 AI 研究工作坊的教师
- 需要统一 AI 使用规范的研究团队

### 核心原则

AI 可以自动化劳动，但不能替代学术责任。

对于严肃研究，建议使用：一个 GitHub 仓库管理研究项目，一个 ChatGPT 或 Claude Project 管理 AI 工作空间，一个 AI 使用日志记录可追溯性，一个清晰的文件夹结构管理代码、数据、论文和结果。

### 点击、复制、直接使用

本仓库的材料应当让读者可以直接点击进入主题，复制可复用说明，粘贴到 ChatGPT、Claude、Codex 或 Cursor 中，并按自己的研究项目调整。页面应尽量让非技术背景的学者也能读懂，并清楚说明输入、输出、风险和核查步骤。
