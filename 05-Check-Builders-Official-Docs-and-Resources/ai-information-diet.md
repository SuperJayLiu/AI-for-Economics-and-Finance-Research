# AI Information Diet

## What Problem This Solves

AI updates are noisy. Scholars need a deliberate information diet so they learn useful workflows without chasing every tool launch.

| Source type | Use it for | Risk |
| --- | --- | --- |
| Official docs | Feature accuracy | May not teach research use |
| Builders | Practical workflows | May be tool-biased |
| Economists using AI | Field relevance | May be anecdotal |
| GitHub repos | Reusable implementation | May be unmaintained |
| Newsletters | Awareness | Can become hype |
| X/LinkedIn | Discovery | Very noisy |

## Recommended Weekly Routine

1. Check official docs only for tools you actually use.
2. Read one or two builders who show workflows, code, failures, or updates.
3. Save useful ideas as resource cards.
4. Test one idea on a small nonconfidential task.
5. Keep or discard the idea based on research value, not novelty.

## Rule

Follow workflows, code, mistakes, and documented updates. Ignore posts that only list tools or promise impossible productivity.

## Sources And Workflow Influences

Draws on builder-oriented AI learning, curated AI libraries, and academic workflow curation.
