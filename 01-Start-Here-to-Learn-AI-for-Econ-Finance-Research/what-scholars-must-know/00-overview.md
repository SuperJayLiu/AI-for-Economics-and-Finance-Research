# What Scholars Must Know

This section explains the minimum concepts economics and finance scholars need before using AI in research.

> [!IMPORTANT]
> If you remember only one idea: an LLM is not a database, not a referee, not an econometrician, and not a coauthor. It is an assistant whose work must be checked.

## Key Points

- LLMs are not databases.
- AI can generate fluent falsehoods.
- Search, reasoning, writing, and coding are different tasks.
- File upload does not guarantee source-grounded answers.
- RAG and agents can help, but they create new verification duties.
- Local models may reduce privacy risk, but they do not remove accuracy risk.

## Quick Self-Test

| Question | If your answer is "no" |
| --- | --- |
| Can I verify the output? | Do not use it in research yet. |
| Is the data safe to upload? | Do not upload it. |
| Do I know whether this is search, reasoning, writing, or coding? | Split the task before asking AI. |
| Can I recover files if AI changes them? | Set up Git first. |
