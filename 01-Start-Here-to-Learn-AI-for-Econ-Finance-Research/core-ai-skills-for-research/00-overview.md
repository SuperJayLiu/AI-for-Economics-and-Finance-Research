# Core AI Skills For Research

This section teaches practical research skills for using AI well.

> [!NOTE]
> Good AI use is not "ask a better prompt." Good AI use is controlled workflow design.

The central skill is converting a vague research task into a controlled workflow with:

- inputs
- context
- rules
- expected output
- verification steps
- record of what changed

Example:

Weak request: "Summarize this paper."

Stronger workflow: ask AI to identify the research question, data, model, identification strategy, main results, limitations, contribution, relation to your project, and items that must be checked in the original text.

## The Research Workflow Formula

```text
Task + Context + Rules + Output format + Verification = usable AI workflow
```

## Example

| Vague task | Controlled workflow |
| --- | --- |
| Help me with this paper | Identify the research question, method, evidence, limitations, and claims I must verify. |
| Fix my Stata code | Explain what each block does, flag possible data or identification errors, and suggest tests before editing. |
| Improve my intro | Preserve all claims and citations, identify vague contribution language, and propose a clearer structure. |
