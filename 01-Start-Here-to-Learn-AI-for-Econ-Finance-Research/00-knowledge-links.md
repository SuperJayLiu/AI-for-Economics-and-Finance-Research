## How To Use These Links

Use official docs for tool behavior. Use economist-facing papers for research implications. Use builder resources for workflow patterns. Keep all tool claims dated.

## Add A New Link Only If It Helps A Workflow

Before adding a resource, ask:

- Does it explain a concept readers need?
- Does it show a workflow, code, template, or failure case?
- Is it official documentation for a tool we discuss?
- Is it relevant to economics, finance, academic writing, empirical coding, data safety, GitHub, or research presentations?
- Can the advice be dated and checked?

If the answer is no, put it aside. The handbook should reduce noise, not collect it.
