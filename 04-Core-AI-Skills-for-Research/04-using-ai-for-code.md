# Using AI For Code

AI can explain, draft, and review code, but generated code must be run and inspected.

Use AI for:

- explaining unfamiliar scripts
- adding comments
- drafting small functions
- checking whether code matches the described empirical design
- finding likely bugs
- writing tests or toy examples

Do not trust code without:

- running it
- checking logs
- inspecting outputs
- testing small examples
- reviewing data transformations
- verifying table construction
