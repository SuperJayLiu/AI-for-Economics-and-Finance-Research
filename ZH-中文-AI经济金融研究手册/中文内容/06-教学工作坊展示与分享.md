# 06 教学、工作坊、展示与分享

这一页适合教师、PI、PhD 组织者、RA 指导者和研讨会组织者，用来教学、培训、练习展示和分享本仓库内容。

> [!TIP]
> 一个好的 AI 工作坊不应该只讲工具，而应该让参与者带走一个自己的项目指令、一个可复制技能和一个核查规则。

## 两次课工作坊

### 第 1 次：基础与风险

目标：让研究者理解 AI 能做什么、不能做什么。

议程：

1. AI 输出不是 evidence。
2. LLM、project、skill、agent、MCP 的区别。
3. 数据安全、保密性和机构规则。
4. 不编造引用的文献综述。
5. 实证方法写作和检查。
6. AI 使用日志和披露。

练习：

```text
选择一个你经常重复的研究任务，把它改写成：
1. purpose
2. 安全输入
3. AI 指令
4. 预期输出
5. 核查清单
```

### 第 2 次：应用工作流

目标：让研究者建立一个安全的 AI 研究工作流。

议程：

1. 一篇论文，一个仓库，一个 AI 项目；
2. GitHub 安全和 `.gitignore`；
3. `AGENTS.md` 和 `CLAUDE.md`；
4. 复现包 agent 工作流；
5. 用 AI 练习展示；
6. 不追逐噪音地跟进更新。

练习：

```text
为一篇论文创建安全 AI 项目设置：
- 项目指令
- 文件夹结构
- 数据规则
- AI-use log
- 第一个要创建的技能
- 第一个核查步骤
```

## 一页规则 handout

```text
1. AI 输出不是 evidence。
2. 不要相信 AI 生成的 citation。
3. 只要 AI 能改文件，就用 Git。
4. 私人、授权、受限和保密数据不要上传到未批准工具。
5. 执行前先计划。
6. 重复任务变成 skill。
7. 保留 AI-use log。
8. 用原始来源检查代码、表格、方程、引用和 claim。
9. 工具比较必须带日期。
10. 只有验证强于自动化时，才自动化。
```

## RA 入门清单

```text
RA 使用 AI 前必须：
- 阅读项目 README；
- 阅读 DATA.md；
- 阅读 AGENTS.md 或 CLAUDE.md；
- 确认数据隐私规则；
- 设置 Git；
- 理解分支工作流；
- 知道哪些文件不能编辑；
- 知道如何记录 AI use；
- 跑一个小的可复现性检查；
- 上传或分享草稿前先问。
```

## 展示练习活动

准备四页：

- 研究问题；
- data/design/model；
- 主要结果；
- 限制条件。

然后让 AI 扮演研讨会听众：

```text
请作为严格但公平的研讨会听众。
基于这四页内容，提出：
1. 10 个常见问题；
2. 5 个尖锐但公平的问题；
3. 5 个澄清性问题；
4. 哪些地方过度声称；
5. 哪些回答需要更多 evidence。
```

## Slides 两种风格

| 类型 | 适合场景 | 注意事项 |
| --- | --- | --- |
| 互动 HTML slides | 教学、线上分享、机制动画、公开解释 | 互动必须帮助理解机制、数据或识别，不要只是装饰。 |
| LaTeX/Beamer slides | 研讨会、会议、求职报告、需要公式的报告 | 不要为了压缩 slide 而删掉关键假设和限制。 |

## 工作坊后的建议

参与者应至少带走：

- 一个适合自己论文的 ChatGPT/Claude Project 指令；
- 一个自己领域的可复用 skill；
- 一个 AI-use log 模板；
- 一个数据安全规则；
- 一个下周可以尝试的低风险任务。
