## 安全获取更新

优先推荐 GitHub 原生方式，因为它不需要维护者额外收集邮箱。

| 方式 | 如何使用 | 信息安全说明 |
| --- | --- | --- |
| GitHub Watch | 在仓库页面点击 Watch，选择通知级别。 | 适合 GitHub 用户，维护者不需要额外收集你的邮箱。 |
| GitHub Releases | 只关注 releases。 | 低频，适合只想知道稳定版本更新的读者。 |
| 邮件更新列表 | 发邮件到 [jay.liu@bristol.ac.uk](mailto:jay.liu@bristol.ac.uk)，标题写 `[AI Econ Finance Updates] Subscribe`。 | 只发送姓名和邮箱即可，不要发送研究数据、论文草稿、审稿材料、学生数据或保密信息。 |

退订：发邮件到同一地址，标题写 `[AI Econ Finance Updates] Unsubscribe`。

邮件列表规则：

- 只用于本仓库更新、版本说明、工作坊信息和重要技能更新；
- 只保存必要联系信息；
- 不向第三方分享邮件列表；
- 不通过订阅邮件接收或处理保密研究材料；
- 如果学校、雇主、数据提供方、期刊、会议或基金有更严格规则，遵守更严格规则。

## 来源使用政策

本仓库简要引用外部资源，只提取与经济学和金融学研究相关的 workflow ideas。不要在 license 不允许时整段复制别人的 skill、代码或写作。

提取规则：

```text
Source -> workflow pattern -> econ/finance adaptation -> copy-ready asset -> verification rule
```

## 如何判断一个外部资源是否值得加入？

加入资源至少应满足一条：

- 教授可复用 workflow；
- 是重要工具的官方文档；
- 与经济学或金融学研究直接相关；
- 提供工作代码或可复现实例；
- 解释风险、限制或失败案例；
- 帮助研究者建立 skill，而不是追逐工具热度。

谨慎或排除：

- “20 个 AI 工具改变你的 PhD”式清单；
- “一小时写完论文”式夸张说法；
- 没有日期的工具排名；
- 忽视隐私、Git 和人工核查的建议；
- 没有真实工作流的意见。

## 关注真正构建工作流的人，而不是只转发消息的人

可以关注：

1. 官方文档，用来确认工具实际行为；
2. 经济学和金融学研究者的真实工作流；
3. 发布可用技能、代码和 agent 设置的人；
4. 有可复现实例的 GitHub repo；
5. 讨论失败案例和限制的社区。

不要只关注：

- 转发新闻的人；
- 没有实际例子的工具排名；
- 不提数据安全和验证负担的建议；
- 永久声称某个模型“最好”的比较。

## 可以借鉴的资源库结构

好的 AI 研究资源页不应该只是长长的链接列表，而应该像一个小型数据库。每个资源最好记录：

| 字段 | 为什么重要 |
| --- | --- |
| 名称和 URL | 让读者可以直接点击 |
| 作者或维护者 | 帮助判断来源和责任 |
| 类别 | 区分 learning material、application tool、setup、official docs、dataset、methods reference |
| 主题 | 标明 workflow、Stata、WRDS、skills/MCP、writing、data、slides、methods 等 |
| 难度 | 帮助新手避免过早使用高级工具 |
| 语言 | 方便维护中文和英文路径 |
| 直接可用价值 | 说明是可复制 skill、可安装工具、可跟随 workflow，还是只适合作背景阅读 |
| 风险提醒 | 记录 license、数据安全、工具版本、幻觉风险 |
| 最后检查日期 | 避免过期工具建议 |
| 应放入本仓库哪里 | 明确应进入哪个 README、skill 或 workflow |

建议保留这些视图：

| 视图 | 用途 |
| --- | --- |
| Start Here | 新手第一批资源 |
| Learning Materials | 教程、讲座、课程和解释性材料 |
| Application Tools | 可以直接试用的 skills、agents、MCP、scripts 和 templates |
| Chinese Resources | 中文或双语资源 |
| Needs Testing | 看起来有用但还没有测试过的资源 |

可复制资源卡片：

```markdown
## [资源名称]

URL:
作者/维护者:
类别: Learning material / Application tool / Setup / Official docs / Dataset / Methods reference
主题:
难度: Beginner / Intermediate / Advanced
语言:
直接可用价值:
风险提醒:
应该放入本仓库哪里:
最后检查日期:
```

这个结构可以借鉴 Gen Li 和 Siyang Liu 维护的 Claude Code economics/finance 资源库：它按类别、主题、难度、语言和用途组织资源，并把 Start Here、Learning Materials、Application Tools 和中文资源分开。这个设计比普通链接列表更适合学术读者。

## 工具比较必须带日期

```text
最后检查日期：
检查过的工具版本或套餐：
测试任务：
作者判断：
已知不确定性：
```

不要写“Codex 最好”或“Claude 最好”。应该写：“截至某日期，在某任务、某约束下，某工具更合适；未来可能因为价格、模型、权限、集成或隐私规则变化而改变。”

## 常用经济金融数据入口和权限提醒

这些链接只是起点，不等于使用许可，也不等于可以把数据上传到 AI 工具。尤其要区分：

```text
网页可访问 != 数据可再分发 != 可以上传到公共 AI 工具 != 可以放进 GitHub
```

在使用 AI 处理任何数据前，先检查：

- 数据提供方 license 或 terms of use；
- 学校、雇主、IRB/伦理审查、基金、期刊、会议规则；
- 数据是否 public、licensed、restricted、confidential、embargoed 或 coauthored；
- public AI tools、cloud tools、MCP connectors、coding agents 是否被允许；
- 是否可以只给 AI metadata、变量字典、toy data、synthetic data 或代码错误信息，而不是原始数据。

## Dataset Decision Table：数据使用决策表

| Dataset/material | Access type | Public AI upload? | 要检查的规则 | 更安全的 AI workflow |
| --- | --- | --- | --- | --- |
| FRED、BEA、BLS、World Bank、IMF、OECD aggregate data | public | 通常可以使用 public series IDs 和 documentation | source citation、revision/vintage、API terms | 让 AI 写 retrieval code 和 metadata checks；记录 IDs 和 dates |
| SEC filings and EDGAR metadata | public | 通常可以使用 filing URLs 和 public text | SEC fair-access guidance 和 parser limits | 记录 CIK、accession、form、date、parser version |
| IPUMS 或 public-use microdata | registered public-use | 不自动等于可以上传 | IPUMS terms、reidentification rules、institution rules | 使用 variable metadata、code、toy rows |
| WRDS/CRSP/Compustat/IBES/OptionMetrics/TAQ | licensed institutional | 通常不要上传 raw extracts | WRDS/vendor license、institution cloud/AI rules | 使用 schemas、variable dictionaries、toy examples、code templates |
| Bloomberg、FactSet、LSEG、TRACE licensed products | commercial/licensed | 通常不要上传 | terminal/API/license/redistribution rules | 使用 metadata 和 synthetic examples；extracts 不进 public AI |
| Administrative or restricted microdata | restricted/confidential | 未明确批准则不要 | DUA、IRB/ethics、secure-environment policy | approved secure environment、synthetic examples、disclosure review |
| Proprietary firm data 或 private transaction data | private/confidential | 未合同允许则不要 | NDA、client、employer、legal、security rules | 使用 schemas 和 toy data；不要粘贴 actual records |
| Referee reports、unpublished drafts、coauthor notes | confidential or coowned | policy 和 consent 允许才可以 | journal/conference/editor/coauthor rules | 使用 local/institution-approved tool 或不暴露文本地提问 |

### 公开宏观、经济和国际数据

| 数据入口 | 适合什么 | 权限与保密提醒 | AI 使用建议 |
| --- | --- | --- | --- |
| [FRED](https://fred.stlouisfed.org/) 和 [FRED API](https://fred.stlouisfed.org/docs/api/fred/) | 宏观、货币、金融、劳动力、政策时间序列 | 公开数据，但 series 定义和 transformation 很重要 | 可以让 AI 写 API 代码；记录 series ID、频率、转换和下载日期 |
| [ALFRED](https://alfred.stlouisfed.org/) | vintage / real-time 经济数据 | 公开 vintage data | 如果研究会受数据修订影响，用它避免 revised-data look-ahead bias |
| [BEA data and API](https://apps.bea.gov/api/signup/) | 国民账户、地区账户、国际账户 | 公开数据，但表格会修订 | 记录 table ID、line number、release date 和 transformation |
| [BLS data and API](https://www.bls.gov/bls/api_features.htm) | 劳动力、价格、生产率、职业数据 | 公开数据；要注意季调、benchmark revisions 和 series 定义 | 可以让 AI 写查询代码，但必须从 BLS 核对 series ID 和定义 |
| [Census Data API](https://www.census.gov/data/developers/data-sets.html) | Census/ACS/经济统计的公开汇总数据 | 公开 API 数据不等于 restricted microdata | 可以让 AI 帮你写 API query；restricted microdata 不要用公共 AI |
| [Federal Reserve Data Download Program](https://www.federalreserve.gov/datadownload/help/) | H.8、H.15、Z.1 等美联储统计发布 | 多数为公开 current-release data；real-time 历史可能需要其他来源 | 记录 release、series、access date，判断是否需要 vintage 数据 |
| [World Bank Indicators API](https://datahelpdesk.worldbank.org/knowledgebase/articles/889392) | 国际发展指标 | 公开指标，但跨国和跨期定义可能不同 | 让 AI 写 API 代码和 metadata 摘要；人工核对 indicator 定义 |
| [IMF Data APIs](https://data.imf.org/en/Resource-Pages/IMF-API) | 国际宏观、国际收支、IFS、WEO 类数据 | 不同数据集访问方式可能不同 | 记录 dataset code、API endpoint、filters 和下载日期 |
| [OECD Data Explorer API](https://www.oecd.org/en/data/insights/data-explainers/2024/09/api.html) | OECD 指标和跨国 panel | 不同数据集可能有不同访问要求 | 引用 dataset、版本、filters 和 API 日期 |
| [NBER Public Use Data Archive](https://www.nber.org/research/data) | public-use 经济、人口和企业数据 | 方便但可能不是最新源数据 | 检查原始来源和 license，不要只依赖镜像 |

### Microdata、restricted data 和 registry

| 数据入口 | 适合什么 | 权限与保密提醒 | AI 使用建议 |
| --- | --- | --- | --- |
| [IPUMS Terms of Use](https://www.ipums.org/about/terms) | Census、ACS、international、health 等 harmonized microdata | 需要注册并遵守 terms；不得 reidentify | public-use 不等于 upload-anywhere；优先用 metadata/toy data |
| [ICPSR confidentiality and restricted-use data](https://www.icpsr.umich.edu/sites/icpsr/about/policies/confidentiality) | 社会科学 public-use 和 restricted-use 数据 | restricted-use 需要申请、DUA 和安全方案 | restricted files 不要用于公共 AI；让 AI 在 synthetic example 上帮忙写代码 |
| [Census Federal Statistical Research Data Centers](https://www.census.gov/about/adrm/fsrdc.html) | 联邦 restricted household、firm、linked employer-employee、administrative data | 只能在批准的安全环境中使用 | AI 使用必须得到数据主管部门和机构明确批准 |
| [World Bank Microdata Library](https://microdata.worldbank.org/index.php/about) | household、firm、facility 和发展调查 microdata | 每个数据集有自己的 access category 和 terms | 逐个阅读 terms，不要假定所有 World Bank microdata 都开放 |
| [AEA RCT Registry](https://www.aeaweb.org/journals/policies/rct-registry) | RCT 注册、protocol、version history、透明度记录 | 公开 metadata 可能和 embargoed/confidential files 并存 | 只对公开 protocol 和计划使用 AI；不要暴露 confidential trial documents |

### 具体数据处理例子

| 数据场景 | AI 可以相对安全地帮什么 | 不要上传什么 | 具体指令 |
| --- | --- | --- | --- |
| FRED 宏观序列 | 查询 metadata、写 API 代码、解释转换 | 若项目笔记敏感，不上传私人笔记 | “用这些公开 FRED series ID 写可复现下载代码，并记录频率、转换和下载日期。” |
| BLS 就业数据 | 写 API query、解释 series 定义、做季调检查清单 | 私人研究笔记 | “先对照 BLS 文档核查每个 series ID，再解释结果。” |
| IPUMS/ACS/CPS public-use microdata | 解释变量字典、写 toy code | 个人层面 extracts，除非条款和机构允许 | “只使用变量名和 metadata，在 synthetic rows 上写代码。” |
| WRDS/CRSP/Compustat | 写 merge plan、link table audit、toy merge test | raw extracts、query results、账号、授权文件 | “只使用 schema 描述和 toy identifiers，不要要求或暴露 WRDS extracts。” |
| EDGAR filings | 写 scraper/parser 方案、XBRL 检查、text-as-data 验证 | 私人标注或未公开 linked dataset | “使用公开 filing URL，并记录 CIK、accession number、form、filing date、parser version。” |
| TRACE/Bloomberg/FactSet/LSEG | 设计代码结构、变量字典、audit plan | 授权 extracts 和终端/API 输出，除非 license 允许 | “使用 metadata 和 synthetic examples，不要把商业数据粘贴到公共 AI。” |
| restricted administrative microdata | 在 synthetic example 上写代码模式、写 disclosure checklist | 真实记录、linked IDs、cell counts、raw outputs | “默认不能外部 AI 上传，先问安全环境和 disclosure rules。” |

可复制数据安全指令：

```text
请作为经济学/金融学项目的数据权限和 AI 安全助手。

数据集：
[名称/提供方]

研究用途：
[简述]

访问状态：
[public/licensed/restricted/confidential/unknown]

我可以安全提供的材料：
[metadata / variable names / toy data / public URLs / code only / other]

在给建议前，请先：
1. 判断数据敏感性；
2. 列出哪些内容不能上传公共 AI；
3. 建议更安全的 AI 输入，例如 metadata、变量字典、toy data 或 synthetic examples；
4. 指出我必须核查的 license、IRB/伦理、学校、期刊、合作者或数据提供方规则；
5. 建议 citation/version/access log。

如果任何术语、规则或权限边界不清楚，请先提出澄清问题。
```

## 资源纳入标准

只有当外部资源至少满足下面一条时，才适合加入本仓库：

- 教读者一个可以重复使用的研究工作流；
- 说明官方工具行为、权限、限制或配置方式；
- 与经济学或金融学研究任务直接相关；
- 提供可以运行、测试或复现的代码、模板或案例；
- 解释风险、失败模式、局限或核查方法；
- 帮助研究者建立技能，而不是追逐工具热度。

需要谨慎标注或暂时不加入的资源：

- 没有测试过程的工具排行榜；
- “一小时写完论文”这类无法核查的承诺；
- 不带日期的模型/工具优劣比较；
- 忽视数据隐私、Git、代码测试、引用核查或机构规则的建议；
- license 不清楚、来源不清楚或无法确认作者的模板。

加入一个资源前，先用下面的卡片判断：


## 如何从 builders 学习

这里的 builders 指真正构建工作流、工具、技能、agent、课程或研究系统的人。阅读这类资源时，不要只记住人名或工具名，而要提取可复用结构。

| 看到的内容 | 应该提取什么 | 不要做什么 |
| --- | --- | --- |
| 某个 Claude Code/Codex workflow | 输入、步骤、批准点、输出、核查方式 | 直接照搬到保密数据或未设 Git 的项目 |
| 某个 skill repo | 任务边界、所需输入、输出格式、失败模式 | 不看 license 就复制全文 |
| 某个工具比较 | 测试任务、日期、价格/权限假设 | 写成永久结论 |
| 某个 AI slides skill | 讲述结构、视觉检查、导出方式 | 让动画盖过研究问题和证据 |
| 某个 agent demo | plan-first、approval gate、diff review、test command | 让 agent 无审批地改论文、数据或代码 |

## 需要持续关注的主题

这些主题变化快，不能写成永久结论。更新时要优先查官方文档、机构规则、期刊政策和实际测试结果。

| 主题 | 为什么重要 | 应关注什么 |
| --- | --- | --- |
| AI disclosure policies | 期刊、会议、基金、学校和雇主规则不同 | 官方政策页、投稿系统说明、editor/data-editor 要求 |
| local/private models | 敏感数据可能只能在本地或机构批准环境使用 | 机构安全指导、模型质量、部署成本、审计记录 |
| AI coding agents | 能力、价格、上下文窗口、权限和 GitHub 集成变化快 | 官方 docs、实际项目测试、GitHub issue 模式 |
| MCP security | 连接器会增加权限和数据暴露面 | MCP 官方文档、安全公告、scope/permission 设置 |
| AI-generated finance research | 论文生产速度可能超过审查标准 | factor mining、HARKing、multiple testing、replication |
| AI for teaching | 课程助手和学习工具会涉及学生隐私 | 学校政策、学生数据保护、学术诚信规则 |
| LLM-generated variables | AI 可能成为 empirical measurement instrument | prompt sensitivity、model-version drift、validation、disclosure |
| AI execution layer | agent 可以清洗数据、写 estimator、生成表图 | 数据 lineage、toy-data tests、可复现日志、人工批准 |
