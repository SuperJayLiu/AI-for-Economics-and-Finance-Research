# 02 复制即用：AI 研究指令与模板

这一页提供中文可复制模板。使用方法：复制一个代码块，把方括号中的内容替换成自己的研究事实，然后要求 AI 先给计划，再执行。

> [!IMPORTANT]
> 不要一次复制很多模板。选择一个明确任务，提供真实背景，要求 AI 标出不确定性，并人工核查。

## 通用安全指令

在任何 serious research 任务前都可以加上：

```text
不要编造引用、数据来源、系数、稳健性检验、制度细节、数学推导或文献结论。
如果信息不足，请明确说明缺少什么，以及我需要人工核查什么。
请区分：已知事实、解释、建议、不确定性。
请遵守学校、雇主、期刊、会议、基金、数据提供方和合作者关于 AI 使用的规则。
如果相关规则比本指令更严格，请遵守更严格的规则。
```

## 研究想法压力测试

```text
请作为一位严格但建设性的经济学/金融学研究导师，压力测试我的研究想法。

研究领域：
[填写领域]

初步想法：
[用一段话描述]

可能数据：
[数据来源、是否能获得、频率、观测单位]

可能机制：
[经济机制或金融机制]

目标读者：
[seminar / PhD field paper / journal / policy audience]

请完成：
1. 用一句精确的话重述研究问题。
2. 识别 outcome、treatment/exposure/shock、观测单位和时间维度。
3. 解释为什么这个问题在经济学或金融学上重要。
4. 列出这个想法可能重要的三个理由。
5. 列出这个想法可能失败的五个理由。
6. 指出最大的数据、识别、测量和创新性风险。
7. 给出两个更窄版本和两个更有野心版本。
8. 告诉我继续之前必须核查什么。

规则：
- 不要编造文献。
- 不要声称创新性，除非有可核查依据。
- 区分“有潜力”“不确定”“必须核查”。
```

## 来源可靠的文献综述

```text
请基于我提供的真实文献和笔记，构建经济学/金融学文献综述。

研究问题：
[填写]

我的项目可能贡献：
[填写]

我提供的材料：
[论文列表、BibTeX、PDF 摘要、阅读笔记]

请完成：
1. 按机制、方法、数据和研究场景分组。
2. 找出最接近我项目的论文，并解释为什么接近。
3. 区分文献已经说明了什么、还没有说明什么。
4. 谨慎说明我的项目可能贡献。
5. 标出“假创新性”风险、遗漏文献风险和引用风险。
6. 起草文献综述大纲。
7. 写一个来源可靠的综合段落。

规则：
- 只使用我提供或明确核查过的文献。
- 不要编造作者、标题、期刊、年份、结果或 citation。
- 不要写“little is known”，除非材料支持。
- 需要引用的句子标注 [需要引用]。
```

## 应用经济学实证方法段落

```text
请为一篇应用经济学论文起草实证方法部分。

输入：
- 研究问题：[填写]
- 制度背景或场景：[填写]
- 观测单位：[填写]
- 样本期间：[填写]
- 数据来源：[填写]
- 结果变量：[填写]
- treatment/exposure/shock：[填写]
- 控制变量：[填写]
- 实证设计：[DiD/event study/RD/IV/RCT/panel FE/synthetic control/其他]
- 主要方程：[填写，如果有]
- 识别假设：[填写]
- 有效性威胁：[填写]
- 推断方法：[标准误、聚类、随机化推断等]

请写：
1. 数据和样本段落；
2. 变量构造段落；
3. 带方程的实证设定；
4. 识别论证；
5. 推断和标准误段落；
6. 有效性威胁和稳健性检验；
7. 系数含义和经济量级解释；
8. 这个设计能说明什么、不能说明什么。

规则：
- 不要发明控制变量、样本限制或稳健性检验。
- 不要把相关关系写成因果关系。
- 信息缺失时写 [需要作者补充]。
```

## 金融学实证方法段落

```text
请为一篇金融学研究论文起草实证方法部分。

输入：
- 领域：[asset pricing/corporate finance/banking/household finance 等]
- 研究问题：[填写]
- 数据来源：[CRSP/Compustat/WRDS/TRACE/Dealscan/13F/EDGAR/proprietary 等]
- 观测单位：[firm-month/stock-day/loan-quarter/fund-month 等]
- 样本期间和筛选规则：[填写]
- 关键变量：[outcomes/predictors/treatment/controls]
- 实证设计：[panel regression/event study/portfolio sorts/Fama-MacBeth/DiD/IV/RD 等]
- 主要方程或组合构造：[填写]
- 收益度量：[raw/excess/abnormal/risk-adjusted]
- 风险调整：[CAPM/FF3/FF5/q-factor/characteristic controls 等]
- 推断方法：[cluster/Newey-West/double clustering/bootstrap]
- 金融学特有风险：[look-ahead/survivorship/delisting returns/liquidity/transaction costs/multiple testing]

请写：
1. 数据构造和样本筛选；
2. 变量定义；
3. 主要设定或 portfolio procedure；
4. 识别或解释边界；
5. 风险调整或 benchmark model；
6. 推断方法；
7. 稳健性和经济量级；
8. 金融学特有 caveats；
9. 复现所需细节。

规则：
- 不要编造筛选规则或 risk factors。
- 不要把 predictability 写成 causal effect。
- 不要忽略 look-ahead、survivorship、delisting、liquidity、transaction cost 和 multiple testing。
```

## 因果识别诊断

```text
请诊断这个经济学/金融学因果设计。

项目事实：
- 研究问题：[填写]
- treatment/exposure/shock：[填写]
- outcome：[填写]
- 观测单位和时间：[填写]
- 数据：[填写]
- 设计：[OLS/panel FE/DiD/IV/RD/synthetic control/event study/其他]
- 比较组或识别变异：[填写]

请检查：
1. 反事实是什么？
2. 哪些变异识别系数？
3. 需要哪些假设？
4. 哪些情况会破坏这些假设？
5. 需要哪些诊断、placebo、falsification 或 robustness checks？
6. 这项设计支持 descriptive、associational、predictive 还是 causal language？
7. 哪些写法应该避免？

返回：
- 设计摘要；
- 假设表；
- 最大威胁；
- 最低限度诊断；
- 安全解释段落。
```

## 理论模型 Discussant

```text
请作为严格但建设性的经济学/金融学 theory discussant 评估这个模型。

模型草稿：
[粘贴模型、命题、证明或 appendix]

背景：
- 领域：[填写]
- 研究问题：[填写]
- 目标机制：[填写]
- 目标贡献：[填写]

请完成：
1. 忠实重构模型：环境、agents、目标函数、约束、timing、信息结构、frictions、equilibrium concept、内生变量和外生变量。
2. 用一两句话隔离核心机制。
3. 区分已经证明的结论、直觉和猜测。
4. 把问题分为 fatal flaws、important weaknesses、optional refinements。
5. 审计关键假设：每个假设在模型中起什么作用，是否必要，是否可弱化。
6. 检查 benchmark 和 limiting cases。
7. 提取经济含义和可能的实证含义。

规则：
- 不要编造证明、假设、机制、引用或结果。
- 不要未经允许把模型改成另一个模型。
- 如果证明不完整，请明确指出还缺什么。
```

## 展示练习

```text
请作为严格但公平的 seminar audience，帮我练习经济学/金融学 presentation。

输入：
- 标题：[填写]
- 摘要：[填写]
- slide outline 或 slides：[粘贴]
- 听众：[field seminar/conference/job talk/PhD workshop]
- 核心 claim：[填写]
- 我最担心的部分：[填写]

请完成：
1. 提出 10 个常见听众问题；
2. 提出 5 个 hostile but fair questions；
3. 提出 5 个 clarification questions；
4. 指出哪里过度声称；
5. 指出缺少哪些制度、数据或识别细节；
6. 起草简洁回答；
7. 建议 slide 修改。

规则：
- 不要奉承。
- 不要编造答案。
- 如果答案需要证据，请说明需要什么证据。
```
