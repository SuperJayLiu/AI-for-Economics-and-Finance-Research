# 05 资料来源、官方文档与更新

这一页说明如何跟进 AI 更新、如何判断外部资料是否值得采用、如何安全订阅更新。

> [!IMPORTANT]
> 不要把这一页当作链接堆积。真正有用的是：从外部资料中提取工作流模式，然后改造成适合经济学和金融学研究的可验证工具。

## 安全获取更新

优先推荐 GitHub 原生方式，因为它不需要维护者额外收集邮箱。

| 方式 | 如何使用 | 信息安全说明 |
| --- | --- | --- |
| GitHub Watch | 在仓库页面点击 Watch，选择通知级别。 | 适合 GitHub 用户，维护者不需要额外收集你的邮箱。 |
| GitHub Releases | 只关注 releases。 | 低频，适合只想知道稳定版本更新的读者。 |
| 邮件更新列表 | 发邮件到 [jay.liu@bristol.ac.uk](mailto:jay.liu@bristol.ac.uk)，标题写 `[AI Econ Finance Updates] Subscribe`。 | 只发送姓名和邮箱即可，不要发送研究数据、论文草稿、审稿材料、学生数据或保密信息。 |

退订：发邮件到同一地址，标题写 `[AI Econ Finance Updates] Unsubscribe`。

邮件列表规则：

- 只用于本仓库更新、版本说明、工作坊信息和重要技能更新；
- 只保存必要联系信息；
- 不向第三方分享邮件列表；
- 不通过订阅邮件接收或处理保密研究材料；
- 如果学校、雇主、数据提供方、期刊、会议或基金有更严格规则，遵守更严格规则。

## 如何判断一个外部资源是否值得加入？

加入资源至少应满足一条：

- 教授可复用 workflow；
- 是重要工具的官方文档；
- 与经济学或金融学研究直接相关；
- 提供工作代码或可复现实例；
- 解释风险、限制或失败案例；
- 帮助研究者建立 skill，而不是追逐工具热度。

谨慎或排除：

- “20 个 AI 工具改变你的 PhD”式清单；
- “一小时写完论文”式夸张说法；
- 没有日期的工具排名；
- 忽视隐私、Git 和人工核查的建议；
- 没有真实工作流的意见。

## 关注真正构建工作流的人，而不是只转发消息的人

可以关注：

1. 官方文档，用来确认工具实际行为；
2. 经济学和金融学研究者的真实工作流；
3. 发布可用技能、代码和 agent 设置的人；
4. 有可复现实例的 GitHub repo；
5. 讨论失败案例和限制的社区。

不要只关注：

- 转发新闻的人；
- 没有实际例子的工具排名；
- 不提数据安全和验证负担的建议；
- 永久声称某个模型“最好”的比较。

## 可以借鉴的资源库结构

好的 AI 研究资源页不应该只是长长的链接列表，而应该像一个小型数据库。每个资源最好记录：

| 字段 | 为什么重要 |
| --- | --- |
| 名称和 URL | 让读者可以直接点击 |
| 作者或维护者 | 帮助判断来源和责任 |
| 类别 | 区分 learning material、application tool、setup、official docs、dataset、methods reference |
| 主题 | 标明 workflow、Stata、WRDS、skills/MCP、writing、data、slides、methods 等 |
| 难度 | 帮助新手避免过早使用高级工具 |
| 语言 | 方便维护中文和英文路径 |
| 直接可用价值 | 说明是可复制 skill、可安装工具、可跟随 workflow，还是只适合作背景阅读 |
| 风险提醒 | 记录 license、数据安全、工具版本、幻觉风险 |
| 最后检查日期 | 避免过期工具建议 |
| 应放入本仓库哪里 | 明确应进入哪个 README、skill 或 workflow |

建议保留这些视图：

| 视图 | 用途 |
| --- | --- |
| Start Here | 新手第一批资源 |
| Learning Materials | 教程、讲座、课程和解释性材料 |
| Application Tools | 可以直接试用的 skills、agents、MCP、scripts 和 templates |
| Chinese Resources | 中文或双语资源 |
| Needs Testing | 看起来有用但还没有测试过的资源 |

可复制资源卡片：

```markdown
## [资源名称]

URL:
作者/维护者:
类别: Learning material / Application tool / Setup / Official docs / Dataset / Methods reference
主题:
难度: Beginner / Intermediate / Advanced
语言:
直接可用价值:
风险提醒:
应该放入本仓库哪里:
最后检查日期:
```

这个结构可以借鉴 Gen Li 和 Siyang Liu 维护的 Claude Code economics/finance 资源库：它按类别、主题、难度、语言和用途组织资源，并把 Start Here、Learning Materials、Application Tools 和中文资源分开。这个设计比普通链接列表更适合学术读者。

## 工具比较必须带日期

```text
最后检查日期：
检查过的工具版本或套餐：
测试任务：
作者判断：
已知不确定性：
```

不要写“Codex 最好”或“Claude 最好”。应该写：“截至某日期，在某任务、某约束下，某工具更合适；未来可能因为价格、模型、权限、集成或隐私规则变化而改变。”

## 常用经济金融数据入口和权限提醒

这些链接只是起点，不等于使用许可，也不等于可以把数据上传到 AI 工具。尤其要区分：

```text
网页可访问 != 数据可再分发 != 可以上传到公共 AI 工具 != 可以放进 GitHub
```

在使用 AI 处理任何数据前，先检查：

- 数据提供方 license 或 terms of use；
- 学校、雇主、IRB/伦理审查、基金、期刊、会议规则；
- 数据是否 public、licensed、restricted、confidential、embargoed 或 coauthored；
- public AI tools、cloud tools、MCP connectors、coding agents 是否被允许；
- 是否可以只给 AI metadata、变量字典、toy data、synthetic data 或代码错误信息，而不是原始数据。

### 公开宏观、经济和国际数据

| 数据入口 | 适合什么 | 权限与保密提醒 | AI 使用建议 |
| --- | --- | --- | --- |
| [FRED](https://fred.stlouisfed.org/) 和 [FRED API](https://fred.stlouisfed.org/docs/api/fred/) | 宏观、货币、金融、劳动力、政策时间序列 | 公开数据，但 series 定义和 transformation 很重要 | 可以让 AI 写 API 代码；记录 series ID、频率、转换和下载日期 |
| [ALFRED](https://alfred.stlouisfed.org/) | vintage / real-time 经济数据 | 公开 vintage data | 如果研究会受数据修订影响，用它避免 revised-data look-ahead bias |
| [BEA data and API](https://apps.bea.gov/api/signup/) | 国民账户、地区账户、国际账户 | 公开数据，但表格会修订 | 记录 table ID、line number、release date 和 transformation |
| [BLS data and API](https://www.bls.gov/bls/api_features.htm) | 劳动力、价格、生产率、职业数据 | 公开数据；要注意季调、benchmark revisions 和 series 定义 | 可以让 AI 写查询代码，但必须从 BLS 核对 series ID 和定义 |
| [Census Data API](https://www.census.gov/data/developers/data-sets.html) | Census/ACS/经济统计的公开汇总数据 | 公开 API 数据不等于 restricted microdata | 可以让 AI 帮你写 API query；restricted microdata 不要用公共 AI |
| [Federal Reserve Data Download Program](https://www.federalreserve.gov/datadownload/help/) | H.8、H.15、Z.1 等美联储统计发布 | 多数为公开 current-release data；real-time 历史可能需要其他来源 | 记录 release、series、access date，判断是否需要 vintage 数据 |
| [World Bank Indicators API](https://datahelpdesk.worldbank.org/knowledgebase/articles/889392) | 国际发展指标 | 公开指标，但跨国和跨期定义可能不同 | 让 AI 写 API 代码和 metadata 摘要；人工核对 indicator 定义 |
| [IMF Data APIs](https://data.imf.org/en/Resource-Pages/IMF-API) | 国际宏观、国际收支、IFS、WEO 类数据 | 不同数据集访问方式可能不同 | 记录 dataset code、API endpoint、filters 和下载日期 |
| [OECD Data Explorer API](https://www.oecd.org/en/data/insights/data-explainers/2024/09/api.html) | OECD 指标和跨国 panel | 不同数据集可能有不同访问要求 | 引用 dataset、版本、filters 和 API 日期 |
| [NBER Public Use Data Archive](https://www.nber.org/research/data) | public-use 经济、人口和企业数据 | 方便但可能不是最新源数据 | 检查原始来源和 license，不要只依赖镜像 |

### 金融、会计和市场数据

| 数据入口 | 适合什么 | 权限与保密提醒 | AI 使用建议 |
| --- | --- | --- | --- |
| [SEC EDGAR APIs](https://www.sec.gov/edgar/sec-api-documentation) | 公司 filings、submission history、XBRL facts | 公开 filings；自动访问要遵守 SEC fair-access 规则 | 可让 AI 写 parser 和抽取方案；记录 CIK、form、filing date、parser version |
| [OpenFIGI](https://www.openfigi.com/) | 证券标识符和 metadata 映射 | 公共 API 工具，但有访问和频率规则 | 适合 identifier mapping；映射结果必须抽样核查 |
| [WRDS](https://wrds-www.wharton.upenn.edu/) | CRSP、Compustat、TAQ、IBES、OptionMetrics 等平台入口 | 机构授权或订阅；通常不能随意再分发 | 未获明确允许前，不要把 raw extracts 或 query results 上传公共 AI |
| [CRSP](https://www.crsp.org/) | 股票收益、价格、delisting returns、indexes、mutual funds | licensed data；常通过 WRDS 访问 | 保护 extracts；记录 delisting、survivorship、linking 和 sample filters |
| [S&P Global Academic Research Essentials / Compustat](https://www.spglobal.com/market-intelligence/en/solutions/products/spglobal-academic-research-essentials) | Compustat fundamentals 和相关学术金融数据 | licensed product；访问和再分发受限 | 默认只给 AI 变量字典或 toy example，除非机构明确允许 |
| [S&P Global Fundamental Financial Data](https://www.spglobal.com/market-intelligence/en/solutions/products/fundamental-data) | 全球公司基本面和 point-in-time 财务数据 | 商业授权数据 | 不要把 raw extracts 放进公共 AI 或公开 GitHub；记录 point-in-time logic |
| [FINRA TRACE Data and Licensing](https://www.finra.org/filing-reporting/trace/data) | 公司债和固定收益交易数据 | 部分产品需要订阅或授权；再传播可能受限 | 使用 AI/cloud 前检查 license；记录 filters、reporting rules 和清洗逻辑 |
| [Bloomberg Professional Data](https://www.bloomberg.com/professional/products/data/) | 市场、reference、news、analytics 数据 | 商业授权，终端和 API 条款严格 | 未获 license 和学校允许，不要上传 extracts 到公共 AI |
| [LSEG Data and Analytics](https://www.lseg.com/en/data-analytics) | Refinitiv/LSEG 市场、基本面、持股、新闻数据 | 商业授权 | 用 metadata 和 toy schema 获取 AI 帮助，避免暴露授权数据 |
| [FactSet Data](https://www.factset.com/data) | 市场、基本面、所有权、预测、analytics 数据 | 商业授权 | 把 extracts 当作 licensed/confidential material 处理 |
| [Open Source Asset Pricing](https://www.openassetpricing.com/) | 金融异象复现和因子研究基础设施 | 公共研究基础设施，仍需看代码和数据条款 | 适合学习 replication discipline 和 anomaly 检验 |

### Microdata、restricted data 和 registry

| 数据入口 | 适合什么 | 权限与保密提醒 | AI 使用建议 |
| --- | --- | --- | --- |
| [IPUMS Terms of Use](https://www.ipums.org/about/terms) | Census、ACS、international、health 等 harmonized microdata | 需要注册并遵守 terms；不得 reidentify | public-use 不等于 upload-anywhere；优先用 metadata/toy data |
| [ICPSR confidentiality and restricted-use data](https://www.icpsr.umich.edu/sites/icpsr/about/policies/confidentiality) | 社会科学 public-use 和 restricted-use 数据 | restricted-use 需要申请、DUA 和安全方案 | restricted files 不要用于公共 AI；让 AI 在 synthetic example 上帮忙写代码 |
| [Census Federal Statistical Research Data Centers](https://www.census.gov/about/adrm/fsrdc.html) | 联邦 restricted household、firm、linked employer-employee、administrative data | 只能在批准的安全环境中使用 | AI 使用必须得到数据主管部门和机构明确批准 |
| [World Bank Microdata Library](https://microdata.worldbank.org/index.php/about) | household、firm、facility 和发展调查 microdata | 每个数据集有自己的 access category 和 terms | 逐个阅读 terms，不要假定所有 World Bank microdata 都开放 |
| [AEA RCT Registry](https://www.aeaweb.org/journals/policies/rct-registry) | RCT 注册、protocol、version history、透明度记录 | 公开 metadata 可能和 embargoed/confidential files 并存 | 只对公开 protocol 和计划使用 AI；不要暴露 confidential trial documents |

### 可复制的数据权限检查清单

```text
使用 AI 处理数据前，先回答：

1. 数据集名称、提供方、版本和下载日期是什么？
2. 数据是 public、licensed、restricted、confidential、embargoed 还是 coauthored？
3. 哪些规则限制再分发、cloud upload、AI use 和 derived outputs？
4. 是否需要 IRB/ethics、DUA 或数据提供方批准？
5. 能否只给 AI metadata、变量字典、toy data 或 synthetic examples？
6. 哪些文件绝不能进入 GitHub 或公共 AI 工具？
7. 论文或 replication package 里如何写 citation、license note 和 access statement？
8. 哪些代码、日志和核查能证明数据处理正确？
```
