# 05 资料来源、官方文档与更新

这一页说明如何跟进 AI 更新、如何判断外部资料是否值得采用、如何安全订阅更新。

> [!IMPORTANT]
> 不要把这一页当作链接堆积。真正有用的是：从外部资料中提取工作流模式，然后改造成适合经济学和金融学研究的可验证工具。

## 安全获取更新

优先推荐 GitHub 原生方式，因为它不需要维护者额外收集邮箱。

| 方式 | 如何使用 | 信息安全说明 |
| --- | --- | --- |
| GitHub Watch | 在仓库页面点击 Watch，选择通知级别。 | 适合 GitHub 用户，维护者不需要额外收集你的邮箱。 |
| GitHub Releases | 只关注 releases。 | 低频，适合只想知道稳定版本更新的读者。 |
| 邮件更新列表 | 发邮件到 [jay.liu@bristol.ac.uk](mailto:jay.liu@bristol.ac.uk)，标题写 `[AI Econ Finance Updates] Subscribe`。 | 只发送姓名和邮箱即可，不要发送研究数据、论文草稿、审稿材料、学生数据或保密信息。 |

退订：发邮件到同一地址，标题写 `[AI Econ Finance Updates] Unsubscribe`。

邮件列表规则：

- 只用于本仓库更新、版本说明、工作坊信息和重要技能更新；
- 只保存必要联系信息；
- 不向第三方分享邮件列表；
- 不通过订阅邮件接收或处理保密研究材料；
- 如果学校、雇主、数据提供方、期刊、会议或基金有更严格规则，遵守更严格规则。

## 如何判断一个外部资源是否值得加入？

加入资源至少应满足一条：

- 教授可复用 workflow；
- 是重要工具的官方文档；
- 与经济学或金融学研究直接相关；
- 提供工作代码或可复现实例；
- 解释风险、限制或失败案例；
- 帮助研究者建立 skill，而不是追逐工具热度。

谨慎或排除：

- “20 个 AI 工具改变你的 PhD”式清单；
- “一小时写完论文”式夸张说法；
- 没有日期的工具排名；
- 忽视隐私、Git 和人工核查的建议；
- 没有真实工作流的意见。

## 关注真正构建工作流的人，而不是只转发消息的人

可以关注：

1. 官方文档，用来确认工具实际行为；
2. 经济学和金融学研究者的真实工作流；
3. 发布可用技能、代码和 agent 设置的人；
4. 有可复现实例的 GitHub repo；
5. 讨论失败案例和限制的社区。

不要只关注：

- 转发新闻的人；
- 没有实际例子的工具排名；
- 不提数据安全和验证负担的建议；
- 永久声称某个模型“最好”的比较。

## 工具比较必须带日期

```text
最后检查日期：
检查过的工具版本或套餐：
测试任务：
作者判断：
已知不确定性：
```

不要写“Codex 最好”或“Claude 最好”。应该写：“截至某日期，在某任务、某约束下，某工具更合适；未来可能因为价格、模型、权限、集成或隐私规则变化而改变。”

## 资源提取卡片

```markdown
## 来源：[名称]

链接：
类型：官方文档 / 经济学工作流 / 技能仓库 / 课程 / 博客 / 论文
难度：入门 / 中级 / 高级

它做了什么：

可复用工作流模式：

对经济金融研究有什么启发：

可以改造成仓库里的什么工具：

不能盲目复制什么：

license 或署名注意事项：

最后检查日期：
```
