# AI Research Workflow Maturity Ladder

## What Problem This Solves

Scholars need to locate where they are in AI adoption before choosing tools. More automation is not automatically better.

| Level | Name | What it means |
| --- | --- | --- |
| 0 | No AI | Traditional research workflow |
| 1 | Chat use | Ask AI questions manually |
| 2 | Structured chat | Use AI with clear context and verification |
| 3 | Project workspace | Use ChatGPT or Claude Projects for a paper or task |
| 4 | GitHub-assisted workflow | AI helps with files, code, writing, and version control |
| 5 | Skills-based workflow | Repeated tasks become reusable skills |
| 6 | Agentic workflow | AI can plan, edit files, run code, and check results |
| 7 | Research operating system | Integrated AI, GitHub, data, writing, updates, and logs |

## Recommended Progression

Move up only when verification improves with the workflow. Do not adopt agents before you can use Git, inspect diffs, and recover files.

## What The Scholar Must Verify

- whether the workflow protects data
- whether outputs are traceable
- whether files can be recovered
- whether results are reproducible
