# Data Safety By Data Type

Different economics and finance data require different AI-use rules.

| Data type | Public AI tools? | Main caution | Safer alternatives |
| --- | --- | --- | --- |
| Public macro data | Usually acceptable | Check source terms and transformations | Upload small excerpts or metadata |
| Public firm-level data | Depends on license | Commercial databases may restrict redistribution | Use variable names and synthetic samples |
| Commercial financial databases | Often restricted | License terms may prohibit upload | Describe structure without raw data |
| Restricted administrative data | No unless explicitly approved | Legal and institutional restrictions | Secure enclave or approved local tools |
| Household survey microdata | Depends on access rules | Re-identification and use restrictions | Aggregate or anonymized examples |
| Transaction-level data | Usually no | High privacy and proprietary risk | Synthetic data for code debugging |
| Banking data | Usually no | Regulatory, contractual, and confidentiality risk | Local approved systems |
| Confidential firm data | No unless approved | Proprietary information | Summaries cleared for sharing |
| Unpublished working papers | Only with permission and policy check | Intellectual priority and confidentiality | Abstracted notes |
| Referee reports | Only if policy permits | Confidential peer review | Manual notes or local approved tools |
| Student data | Usually no | Privacy law and institutional policy | De-identified examples |
| Proprietary code | Depends on license and employer rules | IP leakage | Local tools or minimal snippets |

When uncertain, do not upload the data.
