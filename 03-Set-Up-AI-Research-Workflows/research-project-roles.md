# Research Project Roles

## What Problem This Solves

Projects work better when the AI has a defined role rather than a vague instruction to "help with research."

## Useful Roles

| Role | Use |
| --- | --- |
| Literature Cartographer | Map papers by question, mechanism, method, and evidence |
| Tough Referee | Critique contribution, identification, theory, and clarity |
| Empirical Design Critic | Flag design assumptions and threats |
| Code Auditor | Review Stata, R, Python, MATLAB, or LaTeX workflows |
| Data Documentation Assistant | Help document variables, files, and transformations |
| Replication Assistant | Map files, scripts, tables, and outputs |
| Theory Intuition Checker | Clarify assumptions, notation, and intuition |
| Writing Coach | Improve structure and clarity without changing meaning |
| Seminar Q&A Opponent | Generate likely questions and answer plans |
| Revision Strategy Assistant | Organize referee comments and response plans |
| Conference Tracker | Track deadlines and opportunities, with manual verification |
| Skill Generator | Turn repeated tasks into reusable workflow instructions |

## Rule

Give each role purpose, limits, safe inputs, output format, and verification duties.

## Sources and Workflow Influences

Draws on academic AI project practices, non-coder AI workflow systems, and agent-role design patterns.
