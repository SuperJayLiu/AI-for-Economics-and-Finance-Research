# AI Skills Management

AI skills management is about deciding what to learn, what to ignore, and how to keep your AI workflow useful as tools change.

> [!TIP]
> Do not try to "learn all AI." Build a small skillset around the research tasks you repeat.

This is different from using AI for a specific research task. It asks:

- Which AI capabilities actually improve my research workflow?
- Which tools create privacy or reproducibility risks?
- Which updates are worth adopting?
- Which online advice is hype?
- How do I build a personal AI research system without becoming an AI engineer?

Core principle: learn capabilities, not tool names.

## A Practical Skill Stack

| Skill | Why it matters |
| --- | --- |
| Give context | AI cannot infer your research design from a vague prompt |
| Decompose tasks | Large tasks hide errors |
| Verify output | Research quality depends on checking |
| Use Git | AI-assisted files need recovery and inspection |
| Build projects | Ongoing work needs persistent context |
| Create skills | Repeated tasks should not be re-prompted from scratch |
| Filter updates | Tool hype wastes research time |

## What To Build First

Start with one personal workflow:

- paper-reading note
- empirical code review
- seminar Q&A practice
- literature map
- revision response tracker

Then test it on low-risk material before using it on serious research.
