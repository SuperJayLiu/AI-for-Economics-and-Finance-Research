# How To Read This Handbook

## The Promise

This handbook is not here to make you a prompt collector. It is here to help you become a better AI-assisted scholar.

The goal is practical: build research workflows that are faster, safer, more transparent, and easier to verify.

> [!TIP]
> Do not read every file before doing anything. Read enough to make one research workflow safer, then test it on a small, low-risk task.

## The Wrong Way To Read It

- hunting for magic prompts
- skipping the risk pages
- adopting every new tool
- using AI before knowing how to verify output
- treating polished language as evidence

## The Right Way To Read It

1. Learn the concepts.
2. Choose one real research task.
3. Build a small workflow.
4. Verify the result manually.
5. Record what AI did.
6. Turn repeated work into a reusable instruction or skill.

## Stop Signs While Reading

Pause and go to the risk pages if your use case involves:

- unpublished coauthor drafts
- referee reports or confidential seminar materials
- WRDS/CRSP/Compustat extracts or licensed databases
- administrative, transaction-level, firm-level, student, or household microdata
- AI tools that can edit files, run code, or push changes
- claims about causal effects, investment implications, or policy recommendations

Start with [Responsible Use and Risks](responsible-use-and-risks/00-overview.md) before continuing.

## What To Do After Reading

Pick one workflow:

- literature review
- empirical code review
- seminar presentation
- revision response
- replication package

Then go to [Set Up Agents and Automated Research Workflows](../../../../03-Set-Up-Agents-and-Automated-Research-Workflows/README.md) and create a project setup.
